# MHDDoS GUI工具使用说明

## 简介

MHDDoS GUI是一个功能完备的图形界面工具，提供了所有MHDDoS命令行工具的功能，让您可以通过友好的图形界面进行DDoS攻击配置和管理。

## 功能特性

### ✅ 攻击配置
- **Layer 7 攻击**: 支持26种应用层攻击方法
- **Layer 4 攻击**: 支持31种传输层攻击方法
- **参数配置**: 线程数、RPC、持续时间等可自定义
- **代理支持**: 完整的代理管理和配置
- **反射器支持**: 放大攻击反射器配置

### ✅ 实时监控
- **状态显示**: 目标、方法、运行状态等信息
- **性能统计**: PPS、BPS、总请求数、总字节数
- **进度条**: 直观显示攻击进度
- **时间显示**: 运行时间和剩余时间

### ✅ 工具功能
- **PING**: 服务器ping测试
- **CHECK**: 网站状态检查
- **INFO**: IP地址信息查询
- **TSSRV**: TeamSpeak SRV解析
- **DSTAT**: 系统统计信息

### ✅ 代理管理
- **自动下载**: 从配置的源自动下载代理
- **代理验证**: 自动验证代理可用性
- **代理列表**: 查看和管理代理列表
- **类型支持**: HTTP、SOCKS4、SOCKS5

### ✅ 日志系统
- **实时日志**: 显示所有操作和状态
- **日志级别**: DEBUG、INFO、WARNING、ERROR
- **日志保存**: 支持导出日志到文件
- **彩色显示**: 不同级别的日志使用不同颜色

## 安装要求

### 系统要求
- Windows 10/11 或 Linux 或 macOS
- Python 3.8 或更高版本

### 依赖安装

```bash
# 安装所有依赖
pip install -r requirements.txt

# 如果缺少tkinter（Linux系统）
# Ubuntu/Debian:
sudo apt-get install python3-tk

# CentOS/RHEL:
sudo yum install python3-tkinter

# Fedora:
sudo dnf install python3-tkinter
```

## 使用方法

### 方法1: 使用启动器（推荐）

```bash
python gui_launcher.py
```

启动器会自动检查依赖并启动GUI。

### 方法2: 直接启动

```bash
python gui.py
```

## 界面说明

GUI工具采用标签页设计，包含以下页面：

1. **⚔️ 攻击配置**: 配置和启动攻击
2. **🔧 工具**: 使用各种网络工具
3. **🌐 代理管理**: 管理代理列表
4. **📋 日志**: 查看操作日志

### 攻击配置页面

#### Layer选择
- **Layer 7 (应用层)**: HTTP/HTTPS攻击，需要URL
- **Layer 4 (传输层)**: TCP/UDP攻击，需要IP:PORT

#### 攻击方法
根据选择的Layer自动更新可用的攻击方法列表。

#### 目标配置
- **Layer 7**: 输入完整URL，如 `http://example.com` 或 `https://example.com/path`
- **Layer 4**: 输入 `IP:PORT` 格式，如 `192.168.1.1:80`

#### 攻击参数
- **线程数**: 并发线程数量（建议: 100-1000）
- **RPC**: 每连接请求数（仅Layer 7，建议: 1-10）
- **持续时间**: 攻击持续时间（秒）

#### 代理配置
- **代理类型**: 
  - `0=全部`: 使用所有类型的代理
  - `1=HTTP`: 仅HTTP代理
  - `4=SOCKS4`: 仅SOCKS4代理
  - `5=SOCKS5`: 仅SOCKS5代理
  - `6=随机`: 随机选择类型
- **代理文件**: 代理列表文件名（位于`files/proxies/`目录）

#### 反射器文件（仅Layer 4放大攻击）
- 用于DNS、NTP、MEM等放大攻击方法
- 文件应包含反射器IP地址列表，每行一个IP

## 术语表与专业解释

### 📊 单位术语

#### PPS (Packets Per Second) - 每秒数据包数
- **定义**: 每秒发送的数据包数量
- **含义**: 衡量攻击强度的指标，数值越高表示攻击越密集
- **典型值**: 
  - 轻量攻击: 1,000 - 10,000 PPS
  - 中等攻击: 10,000 - 100,000 PPS
  - 高强攻击: 100,000+ PPS
- **影响因素**: 线程数、RPC、网络带宽、代理质量

#### BPS (Bytes Per Second) - 每秒字节数
- **定义**: 每秒发送的数据字节数量
- **含义**: 衡量攻击带宽消耗的指标
- **单位换算**: 
  - 1 KB/s = 1,024 B/s
  - 1 MB/s = 1,024 KB/s
  - 1 GB/s = 1,024 MB/s
- **典型值**:
  - 轻量: 几MB/s
  - 中等: 几十MB/s到几百MB/s
  - 高强: GB/s级别

#### RPC (Requests Per Connection) - 每连接请求数
- **定义**: 每个TCP连接发送的HTTP请求数量
- **适用范围**: **仅Layer 7攻击**
- **作用**: 
  - 增加单个连接的利用率
  - 提高攻击效率
  - 减少连接建立开销
- **建议值**:
  - 普通目标: 1-5 RPC
  - 高带宽目标: 5-20 RPC
  - 最大: 100 RPC（不推荐，可能导致连接超时）
- **注意事项**: RPC过高可能导致连接过早关闭

#### 线程数 (Threads)
- **定义**: 同时运行的攻击线程数量
- **作用**: 控制并发攻击规模
- **选择原则**:
  - Layer 7: 100-1,000线程（推荐）
  - Layer 4: 500-5,000线程（推荐）
  - 最大: 10,000线程（取决于系统资源）
- **影响因素**: CPU核心数、内存、网络带宽
- **警告**: 线程数过高可能导致系统资源耗尽

### ⚔️ Layer 7 攻击方法详解

#### 基础方法

**GET**
- **原理**: 发送大量HTTP GET请求
- **特点**: 最基础的HTTP攻击方法
- **适用**: 任何HTTP服务器
- **优势**: 简单高效，资源消耗低
- **绕过能力**: 弱

**POST**
- **原理**: 发送大量HTTP POST请求，包含数据负载
- **特点**: 比GET消耗更多服务器资源
- **适用**: 有表单处理的网站
- **优势**: 占用服务器处理资源更多
- **负载**: 包含JSON数据负载

**HEAD**
- **原理**: 发送HTTP HEAD请求（仅请求头，不请求内容）
- **特点**: 消耗带宽较小但占用连接
- **适用**: HTTP服务器
- **优势**: 服务器仍需处理请求

#### 绕过防护方法

**CFB (CloudFlare Bypass)**
- **原理**: 使用cloudscraper绕过CloudFlare防护
- **特点**: 自动处理验证和cookie
- **适用**: 使用CloudFlare CDN的网站
- **优势**: 可以绕过基本的CloudFlare保护
- **限制**: 无法绕过高级验证（如5秒盾）

**CFBUAM (CloudFlare Under Attack Mode Bypass)**
- **原理**: 等待CloudFlare挑战并尝试绕过
- **特点**: 处理Under Attack模式
- **适用**: CloudFlare的Under Attack模式
- **优势**: 针对高防护模式
- **限制**: 速度较慢，需要等待验证

**DGB (DDoS-Guard Bypass)**
- **原理**: 模拟浏览器行为绕过DDoS-Guard
- **特点**: 自动处理cookie和验证流程
- **适用**: 使用DDoS-Guard服务的网站
- **优势**: 专门针对DDoS-Guard优化

**AVB (Arvan Cloud Bypass)**
- **原理**: 针对Arvan Cloud防护的绕过技术
- **特点**: 适配Arvan Cloud的防护机制
- **适用**: 使用Arvan Cloud的网站
- **优势**: 针对特定CDN服务

**OVH (OVH Bypass)**
- **原理**: 绕过OVH防火墙的检测
- **特点**: 使用特殊请求头和处理方式
- **适用**: OVH托管的主机
- **优势**: 针对OVH防护优化

**BYPASS**
- **原理**: 通用绕过方法，使用Session保持
- **特点**: 适用于多种基础防护
- **适用**: 有简单防护的网站
- **优势**: 通用性好

**GSB (Google Project Shield Bypass)**
- **原理**: 绕过Google Project Shield防护
- **特点**: 针对Google的DDoS防护服务
- **适用**: 使用Google Project Shield的网站
- **优势**: 专门针对Google防护

#### 特殊攻击方法

**STRESS**
- **原理**: 发送大负载的POST请求（524字节数据）
- **特点**: 消耗更多带宽和处理资源
- **适用**: 需要处理数据的服务器
- **优势**: 同时占用带宽和CPU

**SLOW (Slowloris)**
- **原理**: 保持连接打开，缓慢发送数据头
- **特点**: 占用服务器连接池
- **适用**: Apache等有限连接数的服务器
- **优势**: 低带宽消耗，高连接占用
- **资源消耗**: 低带宽，高连接数

**RHEX (Random HEX)**
- **原理**: 在路径中使用随机HEX字符
- **特点**: 增加缓存失效
- **适用**: 有CDN缓存的网站
- **优势**: 绕过缓存，直接攻击源站

**STOMP**
- **原理**: 绕过chk_captcha验证
- **特点**: 使用特殊字符和路径
- **适用**: 有验证码挑战的网站
- **优势**: 处理验证码流程

**DYN (Dynamic)**
- **原理**: 使用随机子域名请求
- **特点**: 绕过基于主域名的防护
- **适用**: 有子域名解析的网站
- **优势**: 可能绕过某些基于域名的限制

**NULL**
- **原理**: 使用空的User-Agent和Referer
- **特点**: 模拟异常请求
- **适用**: 检测简单User-Agent过滤的防护
- **优势**: 绕过基础的UA检查

**COOKIE**
- **原理**: 发送随机Cookie值
- **特点**: 触发PHP的`isset($_COOKIE)`检查
- **适用**: 使用Cookie验证的PHP应用
- **优势**: 消耗服务器Cookie处理资源

**PPS (Packet Per Second)**
- **原理**: 仅发送`GET / HTTP/1.1\r\n\r\n`
- **特点**: 最简化的请求，最大化PPS
- **适用**: 快速发送大量请求
- **优势**: 极高PPS，低资源消耗

**EVEN**
- **原理**: GET方法配合读取响应
- **特点**: 保持连接活跃，读取响应
- **适用**: 需要保持连接的服务器
- **优势**: 占用连接和带宽

**APACHE**
- **原理**: 使用Range请求头攻击Apache漏洞
- **特点**: 触发Apache的Range处理漏洞
- **适用**: 未打补丁的Apache服务器
- **优势**: 可能导致服务器高负载

**XMLRPC**
- **原理**: 攻击WordPress的XMLRPC接口
- **特点**: 使用pingback.ping方法
- **适用**: 启用了XMLRPC的WordPress站点
- **优势**: 可以利用WordPress的放大效应

**BOT**
- **原理**: 模拟搜索引擎爬虫
- **特点**: 使用Google、Bing等爬虫User-Agent
- **适用**: 信任搜索引擎的网站
- **优势**: 可能绕过基础的爬虫检测

**DOWNLOADER**
- **原理**: 缓慢读取下载内容
- **特点**: 保持连接打开，缓慢接收数据
- **适用**: 有下载服务的服务器
- **优势**: 占用连接和带宽

**KILLER**
- **原理**: 使用极多线程快速攻击
- **特点**: 每个线程再启动多个子线程
- **适用**: 需要极高并发的情况
- **警告**: 资源消耗极大，可能导致系统崩溃

**TOR**
- **原理**: 通过Tor2Web网关访问.onion站点
- **特点**: 支持Tor隐藏服务攻击
- **适用**: .onion域名站点
- **优势**: 可以攻击Tor网络中的服务

**BOMB**
- **原理**: 使用bombardier工具进行HTTP/2攻击
- **特点**: 需要安装bombardier外部工具
- **适用**: 支持HTTP/2的服务器
- **优势**: HTTP/2多路复用，效率更高
- **要求**: 需要代理，需要安装bombardier

### 🔥 Layer 4 攻击方法详解

#### TCP/UDP洪水

**TCP**
- **原理**: 建立大量TCP连接并发送随机数据
- **特点**: 消耗目标连接数和带宽
- **适用**: 任何TCP服务（HTTP、HTTPS、SSH等）
- **优势**: 通用性强，有效果
- **资源消耗**: 高带宽，高连接数

**UDP**
- **原理**: 发送大量UDP数据包
- **特点**: 无连接，快速发送
- **适用**: DNS、游戏服务器等UDP服务
- **优势**: 无需建立连接，速度快
- **资源消耗**: 高带宽

**OVH-UDP**
- **原理**: UDP洪水配合随机HTTP头，绕过OVH防护
- **特点**: 专门针对OVH的UDP防护
- **适用**: OVH托管的UDP服务
- **优势**: 绕过OVH的UDP过滤

**SYN**
- **原理**: SYN洪水攻击，发送大量SYN包不完成握手
- **特点**: 占用目标连接队列
- **适用**: TCP服务
- **优势**: 低带宽，高连接占用
- **要求**: 需要原始套接字权限（管理员/root）
- **资源消耗**: 低带宽，高连接数

**ICMP**
- **原理**: ICMP洪水（Ping洪水）
- **特点**: 网络层攻击
- **适用**: 任何网络主机
- **优势**: 绕过应用层防护
- **要求**: 需要原始套接字权限
- **资源消耗**: 中等带宽

#### 放大攻击方法（需要反射器）

**DNS (DNS Amplification)**
- **原理**: 伪造源IP向DNS服务器查询，导致DNS服务器向目标发送大量响应
- **放大倍数**: 10-50倍
- **适用**: 有开放DNS反射器的环境
- **要求**: 
  - 需要反射器文件（开放DNS服务器IP列表）
  - 需要原始套接字权限
  - 需要IP欺骗能力（需要可欺骗IP的主机）
- **警告**: 这是放大攻击，需要特殊网络环境

**NTP (NTP Amplification)**
- **原理**: 利用NTP的monlist功能放大攻击
- **放大倍数**: 200-1000倍
- **适用**: 有开放NTP服务器的环境
- **要求**: 同DNS放大攻击
- **警告**: 放大倍数极高，谨慎使用

**MEM (Memcached Amplification)**
- **原理**: 利用未保护的Memcached服务器放大攻击
- **放大倍数**: 10,000-50,000倍
- **适用**: 有开放Memcached服务器的环境
- **要求**: 同DNS放大攻击
- **警告**: 放大倍数极高，需要严格控制

**RDP (Remote Desktop Protocol Amplification)**
- **原理**: 利用RDP协议放大攻击
- **放大倍数**: 中等
- **适用**: 有开放RDP服务的环境
- **要求**: 同DNS放大攻击

**CHAR (Chargen Amplification)**
- **原理**: 利用Chargen服务放大攻击
- **放大倍数**: 中等
- **适用**: 有开放Chargen服务的环境
- **要求**: 同DNS放大攻击

**CLDAP (Connectionless LDAP Amplification)**
- **原理**: 利用CLDAP协议放大攻击
- **放大倍数**: 50-70倍
- **适用**: 有开放CLDAP服务的环境
- **要求**: 同DNS放大攻击

**ARD (Apple Remote Desktop Amplification)**
- **原理**: 利用Apple Remote Desktop协议放大攻击
- **放大倍数**: 中等
- **适用**: 有ARD服务的环境
- **要求**: 同DNS放大攻击

#### 游戏协议攻击

**MINECRAFT**

- **特点**: 针对Minecraft服务器协议

**MCPE (Minecraft PE)**

**FIVEM**

**FIVEM-TOKEN**

#### 连接攻击

**CPS (Connections Per Second)**
- **原理**: 快速建立和关闭连接
- **特点**: 通过代理快速建立连接
- **适用**: 需要连接管理的服务器
- **优势**: 占用连接资源
- **要求**: 需要代理

**CONNECTION**
- **原理**: 建立连接并保持存活
- **特点**: 占用连接池
- **适用**: 有限连接数的服务器
- **优势**: 长时间占用连接
- **要求**: 需要代理

### 🎯 适用范围总结

#### Layer 7 攻击适用场景
- ✅ HTTP/HTTPS网站
- ✅ Web应用程序
- ✅ 有CDN的网站（可使用绕过方法）
- ✅ 有防护的网站（可使用专用绕过方法）
- ❌ 无法访问HTTP端口的服务

#### Layer 4 攻击适用场景
- ✅ 任何TCP/UDP服务
- ✅ 游戏服务器
- ✅ DNS、邮件等服务
- ✅ 放大攻击（需要特殊环境）
- ❌ 需要管理员权限的方法（SYN、ICMP等）

#### 代理增强场景
- ✅ 本机带宽不足时增加攻击带宽
- ✅ 隐藏本机IP地址
- ✅ 绕过基于IP的限制
- ✅ 分布式攻击（使用多个代理IP）

#### 放大攻击特殊要求
- ⚠️ 需要可IP欺骗的网络环境
- ⚠️ 需要开放反射器的服务器列表
- ⚠️ 需要原始套接字权限
- ⚠️ **警告**: 放大攻击可能造成严重后果，仅用于授权的渗透测试

### 📝 方法选择建议

#### 针对有CloudFlare的网站
1. 首选: **CFB** 或 **CFBUAM**
2. 备选: **BYPASS**、**OVH**
3. 配合: 使用代理增加带宽

#### 针对普通网站
1. 首选: **GET** 或 **POST**
2. 增强: 使用 **STRESS** 增加负载
3. 配合: 高线程数 + 代理

#### 针对游戏服务器
1. 使用对应的协议方法: **MINECRAFT**、**FIVEM**、**TS3** 等
2. 使用高线程数
3. 可以配合代理

#### 针对高带宽目标
1. Layer 4: **TCP**、**UDP**、**SYN**
2. Layer 7: **STRESS**、**POST**
3. 必须使用代理增加带宽
4. 使用高线程数

## 注意事项

### ⚠️ 重要提示

1. **合法使用**: 仅用于授权的安全测试，未经授权使用可能违法
2. **管理员权限**: 某些Layer 4方法（如SYN、ICMP）需要管理员/root权限
3. **代理质量**: 使用高质量代理可以获得更好的攻击效果
4. **线程数建议**: 
   - Layer 7: 100-1000线程
   - Layer 4: 500-5000线程
5. **带宽限制**: 本机带宽不足时，使用代理可以显著增加攻击带宽

### 🔒 权限要求

- **Windows**: 需要以管理员身份运行以支持原始套接字（SYN、ICMP等）
- **Linux/macOS**: 需要以root权限运行以支持原始套接字

### 📝 文件要求

- `files/useragent.txt`: User-Agent列表（必须）
- `files/referers.txt`: Referer列表（必须）
- `files/proxies/*.txt`: 代理列表（可选）
- `files/*.txt`: 反射器文件（仅放大攻击需要）

## 故障排除

### 问题1: GUI无法启动

**解决方案**:
1. 检查Python版本: `python --version`（需要3.8+）
2. 检查依赖: `pip install -r requirements.txt`
3. 检查tkinter: `python -c "import tkinter"`

### 问题2: 攻击无法启动

**解决方案**:
1. 检查目标地址格式是否正确
2. 检查必要的文件是否存在（useragent.txt, referers.txt）
3. 检查代理文件是否正确（如果使用代理）
4. 查看日志页面的错误信息

### 问题3: 代理验证失败

**解决方案**:
1. 检查网络连接
2. 尝试使用不同的代理类型
3. 检查代理源是否可访问
4. 手动添加代理到文件

### 问题4: 原始套接字错误

**解决方案**:
- Windows: 以管理员身份运行程序
- Linux/macOS: 使用sudo运行: `sudo python gui.py`
